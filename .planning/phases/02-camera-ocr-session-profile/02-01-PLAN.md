<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| device (camera) → app | Camera image input is untrusted text; OCR outputs may contain PII — Phase 2 keeps processing local and never transmits images (no INTERNET permission added here) |
| user input → normalization | Manual ingredient entry may include malformed strings; normalizer must sanitize and return UNRESOLVED when unsure |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-2-01 | Information Disclosure | Camera → network | mitigate | AndroidManifest must NOT gain INTERNET; verify no network libs; ML Kit bundled model used (offline). Camera permission is allowed but network egress must be prevented. |
| T-2-02 | Tampering | ImageProxy not closed | mitigate | OcrRepository must always call image.close() in addOnCompleteListener (pattern enforced in action). |
| T-2-03 | Information Disclosure | Manual input contains PII | accept | Manual input remains local and ephemeral; no persistence or transmission; user informed via UI about ephemeral local-only processing. |
</threat_model>
