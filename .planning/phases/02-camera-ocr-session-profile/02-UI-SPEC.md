## Typography

All sizes use Material3 typography roles from `MaterialTheme.typography`. Map roles to concrete values below so the executor does not need to infer defaults.

**Permitted weights: Regular (400) and SemiBold (600) only. No other weights are allowed in Phase 2.**

| Role | Size | Weight | Line Height | Usage in Phase 2 |
|------|------|--------|-------------|-----------------|
| Body | 14sp | Regular (400) | 1.5 | OCR status messages, manual input hint text, dropdown label text |
| Label | 12sp | Regular (400) | 1.3 | Section labels above dropdowns ("Pregnancy Status"), chip labels |
| Heading | 20sp | SemiBold (600) | 1.2 | Screen title ("Scan Label", "Your Profile"), permission rationale heading |
| Display | 28sp | SemiBold (600) | 1.1 | Not used in Phase 2 |

Notes:
- Use `MaterialTheme.typography.bodyMedium` for 14sp/Regular
- Use `MaterialTheme.typography.labelSmall` for 12sp/Regular
- Use `MaterialTheme.typography.titleLarge` for 20sp/SemiBold
- Do NOT declare custom TextStyle objects — use Material3 named roles exclusively
- Medium (500) and Bold (700) weights are not permitted in this phase

## Color

This app uses a dark-first scheme appropriate for a camera-centric scanner. The camera viewfinder is dark by necessity; the rest of the app inherits that dark background for visual consistency.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0F0F0F` (surface — near-black) | Camera screen background, app background, modal scrim behind camera |
| Secondary (30%) | `#1E1E1E` (surfaceVariant) | Session profile card background, DropdownMenu container, manual input card background |
| Accent (10%) | `#4CAF82` (teal-green) | Capture button fill, active selection highlight in DropdownMenu, OCR-complete status chip |
| Destructive | `#CF6679` (Material3 error tone) | Permission permanently denied state, OCR hard-failure message text |

Accent reserved for:
1. Capture button (FAB) fill color — the single primary action on CameraScreen
2. Active/selected item in DropdownMenu for each session selector
3. OCR pipeline success state chip ("Extracted N ingredients")

Never apply accent to: body text, background surfaces, inactive dropdown items, or loading indicators.

Color contract rationale: PRIV-01/PRIV-03 mandate no cloud connectivity — the dark theme visually reinforces the "on-device" trust story and matches the camera preview naturally. Source: design decision (no upstream artifact specified color; dark-first is the sensible default for a scanner app).

MaterialTheme wiring (executor must set up ColorScheme):
```
primary = #4CAF82
background = #0F0F0F
surface = #0F0F0F
surfaceVariant = #1E1E1E
onPrimary = #FFFFFF
onBackground = #E8E8E8
onSurface = #E8E8E8
onSurfaceVariant = #C0C0C0
error = #CF6679
onError = #FFFFFF
```

## Screen Contracts

### Screen 1: CameraScreen

**Route:** `CAMERA` (introduce navigation-compose in Phase 2 — see RESEARCH.md Open Question 1)

**Focal point:** Capture FAB — the single primary action on this screen.

**Layout:**

```
┌─────────────────────────────────────────────────┐
│                                                 │
│          AndroidView { PreviewView }            │
│          fillMaxSize — full bleed               │
│                                                 │
│                                                 │
│  ┌──────────────────────────────────────────┐  │
│  │  [OCR status chip — top of preview]      │  │
│  └──────────────────────────────────────────┘  │
│                                                 │
│                                                 │
│              ●  Capture FAB                     │
│         (56dp diameter, accent fill)            │
│              bottom-center, 48dp above          │
│              system nav bar safe inset          │
└─────────────────────────────────────────────────┘
```

**States:**

| State | What the user sees |
|-------|--------------------|
| Permission not yet requested | Full-screen permission rationale view (no camera preview): heading "Camera access needed", body "SafeGlow needs your camera to scan ingredient labels.", primary button "Allow Camera" (accent fill). No camera preview visible. |
| Permission denied (can re-request) | Same rationale view, body changes to "Tap below to grant camera access in Settings." Button label: "Open Settings" |
| Permission permanently denied | Error color body text: "Camera permission denied. Go to Settings > Apps > SafeGlow Edge > Permissions to enable." No button — user must leave the app. |
| Camera ready (preview active) | Full-bleed PreviewView. Capture FAB visible. No OCR chip. |
| Capturing | Capture FAB shows CircularProgressIndicator (white, 24dp) replacing the camera icon. FAB is disabled (no second tap). |
| OCR processing | FAB remains disabled. Status chip appears at top of preview: "Extracting text..." (loading state, no accent). |
| OCR success | Status chip updates to: "Extracted {N} ingredients" with accent background. FAB re-enables. After 1500ms auto-dismiss chip. If N=0, show error chip instead. |
| OCR zero results | Status chip: "No ingredients found — try manual input" (error color text, surfaceVariant background). Manual input button appears below FAB: "Enter manually" (outlined, accent text). |
| OCR error | Status chip: "Scan failed — retake or enter manually" (error color). Manual input button appears. FAB re-enables for retake. |

**Interaction rules:**
- Tap-to-capture: single tap on FAB only. No tap-to-focus on viewfinder in Phase 2 (defer to Phase 5 polish if needed).
- Back navigation: system back returns to previous screen (or exits if CameraScreen is the root destination).
- Orientation: lock to portrait (`android:screenOrientation="portrait"` on Activity) — simplifies CameraX rotation handling for Phase 2; landscape support deferred.

### Screen 2: ProfileScreen

**Route:** `PROFILE`

**Focal point:** Session context card — the three dropdowns that personalize safety insights.

**Layout:**

```
┌─────────────────────────────────────────────────┐
│  ← Back                                         │
│                                                 │
│  Your Profile                    (Heading)      │
│                                                 │
│  ┌───────────────────────────────────────────┐  │
│  │  Session context  (Label, sm padding top) │  │
│  │                                           │  │
│  │  Pregnancy Status            (Label)      │  │
│  │  [ Exposed DropdownMenu          ▼ ]      │  │
│  │                                           │  │
│  │  Country                     (Label)      │  │
│  │  [ Exposed DropdownMenu          ▼ ]      │  │
│  │                                           │  │
│  │  Skin Concern                (Label)      │  │
│  │  [ Exposed DropdownMenu          ▼ ]      │  │
│  └───────────────────────────────────────────┘  │
│                                                 │
│  ⚠ Session clears when you close the app.      │
│    (Body, onSurfaceVariant color, 12sp)         │
│                                                 │
└─────────────────────────────────────────────────┘
```

**Component choices:**

Use `ExposedDropdownMenuBox` + `ExposedDropdownMenu` + `DropdownMenuItem` from Material3. NOT `DropdownMenu` (requires manual anchor positioning). Source: Material3 ExposedDropdownMenu is the standard for form-style selectors.

**Dropdown options:**

| Field | Options |
|-------|---------|
| Pregnancy Status | Not set, Pregnant, Not pregnant |
| Country | Not set, EU, US, China, Japan |
| Skin Concern | Not set, Normal, Sensitive, Dry, Oily |

"Not set" is always the first item and maps to the `NOT_SET` enum value. When `NOT_SET`, the field shows the field label as placeholder text (e.g., "Select pregnancy status") in `onSurfaceVariant` color.

**States:**

| State | What the user sees |
|-------|--------------------|
| All fields NOT_SET | All three dropdowns show placeholder text in onSurfaceVariant. Session note visible. |
| Field selected | Dropdown shows selected value in onSurface color. Selected item in the open menu has accent background. |

**Session cleared note:** A single `Text` composable below the card. Text: "This profile clears when you close the app. No personal data is stored." Size: 12sp, color: `MaterialTheme.colorScheme.onSurfaceVariant`. Aligns with PROF-03 — must be visible without scrolling on all screens 5.0" and larger.

### Screen 3: ManualInputScreen (inline, not a separate route)

Manual input is NOT a separate navigation destination. It appears as a bottom sheet (`ModalBottomSheet`) launched from CameraScreen when the user taps "Enter manually" in the OCR zero-results or OCR error state.

**Focal point:** Ingredient list TextField — the single input the user must complete to proceed.

**Layout:**

```
┌─────────────────────────────────────────────────┐  ← drag handle
│  Enter Ingredients                (Heading 20sp) │
│                                                 │
│  ┌───────────────────────────────────────────┐  │
│  │  Paste or type the ingredients list       │  │
│  │  (multiline TextField, minLines=4,        │  │
│  │   maxLines=12, fillMaxWidth)              │  │
│  │                                           │  │
│  │  Hint: "AQUA, GLYCERIN, NIACINAMIDE..."   │  │
│  └───────────────────────────────────────────┘  │
│                                                 │
│  [ Analyze Ingredients ]   (Button, accent fill) │
│  [ Keep scanning ]         (TextButton, no fill) │
│                                                 │
└─────────────────────────────────────────────────┘
```

**Rules:**
- `maxLength` enforced at 1000 characters (RESEARCH.md security: prevents OOM from pathological input).
- Show character count: `{n}/1000` in 12sp onSurfaceVariant below the TextField, right-aligned. Updates on every keystroke.
- "Analyze Ingredients" button is disabled when the TextField is blank or contains only whitespace.
- "Analyze Ingredients" button triggers the same `INCINormalizer` pipeline as the OCR path (SCAN-02 requirement).
- "Keep scanning" dismisses the bottom sheet. The camera preview remains paused behind the sheet.

## Interaction Contracts

### Navigation

Introduce `androidx.navigation:navigation-compose` in Phase 2 (RESEARCH.md Open Question 1 recommends this). Two routes:
- `CAMERA` — default start destination
- `PROFILE` — accessible via icon button in CameraScreen top bar

No `ResultsScreen` route yet — Phase 5 adds that.

### Back stack behavior

- CameraScreen → ProfileScreen: ProfileScreen pushed onto back stack. Back presses return to CameraScreen.
- CameraScreen → ManualInput bottom sheet: bottom sheet is modal, not a navigation destination. Dismiss via drag or "Keep scanning" button.

### Scan → Profile linking

A profile icon button in the top-right of CameraScreen navigates to ProfileScreen. Use `Icons.Default.Person` from Material Icons. Size: 24dp. Touch target: 48dp (pad with 12dp on each side via `Modifier.clickable` or `IconButton`). `contentDescription = "Open profile"`.

### Loading / progress

No full-screen loading overlay in Phase 2. Progress is communicated via:
1. Capture FAB spinner (replaces icon during capture)
2. OCR status chip (text state changes)

Do NOT show `CircularProgressIndicator` or `LinearProgressIndicator` as standalone elements in Phase 2 — pipeline progress UI is Phase 5 (UI-01).

### Accessibility

- Capture FAB: `contentDescription = "Capture product label"` on the FAB icon
- Profile icon button (CameraScreen top bar): `contentDescription = "Open profile"`
- OCR status chip: `semantics { liveRegion = LiveRegionMode.Polite }` so screen readers announce state changes
- DropdownMenu items: use `DropdownMenuItem(text = { Text(...) })` — Material3 handles selection semantics
- Back button in ProfileScreen: `contentDescription = "Back"` on the navigation icon
- Manual input TextField: `label = { Text("Ingredient list") }` provides accessible label
